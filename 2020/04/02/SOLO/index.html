SOLO

本文提出了一个新的、简洁的图像实例分割方法。与密集预测的任务相比（如语义分割），实例的个数是任意的，这使得实例分割非常有挑战性。为了对每个实例都预测一个 mask，主流的方法要么延续了“先检测后分割”的策略（如 Mask R-CNN），要么采用聚类的方法将像素点聚合为各个实例。本文则从一个全新的角度来看待实例分割问题，提出了“实例类别”的概念，根据实例的位置和大小，对每个实例的像素点赋予一个类别，巧妙地将实例掩码分割问题转换为一个分类问题。这样，实例分割就被分解成了两个分类任务。本文证明了该方法更加简单、灵活，性能也更好，它可以取得与 Mask RCNN 相似的准确率，超越其它的单阶段实例分割算法。

3.1. Problem Formulation

给定一张图片，实例分割方法需要判断它里面是否存在语义物体的实例，如果有，该方法需要返回分割掩码。SOLO 的核心思想就是将实例分割问题重新表示为两个同时发生的子问题：类别预测和实例掩码生成。具体点就是，该方法将输入图像划分为若干一致的网格，即 $S\times S$。如果物体的中心落在某网格内，该网格就要负责：1）预测语义类别，2）分割该物体实例。

3.1.1 Semantic Category

对于每个网格，SOLO 预测一个 $C$ 维度的输出，表示语义类别的概率，$C$ 是类别的个数。这些概率值取决于这些网格。如果我们将图像划分为 $S\times S$ 个网格，则输出空间为 $S\times S\times C$，如下图中上分支所示。这样设计是基于一个假定，即每一个网格都属于一个单独的实例，因此只属于一个语义类别。在推理时，$C$ 维度的输出表示每个实例的类别概率。

3.1.2 Instance Mask

与语义类别预测平行，每个 positive 网格都会输出一个相应的实例掩码。给定一个输入图片 $I$，我们将之划分为 $S\times S$ 个网格，总共就有至多 $S^2$ 个预测掩码。在3D 输出张量中，我们在第三个维度（通道）中直接对这些掩码进行编码。输出的实例掩码的维度就是 $H_I\times W_I \times S^2$。第 $k$ 个通道负责在第 $(i,j)$ 个网格中分割实例，$k=i\cdot S+j$（ $i$，$j$ 都是从0开始）。这样，我们就在语义类别和掩码之间构建起了如下图所示的一一对应的关系。

构建实例分割。 在 SOLO 中，类别预测和掩码很自然地就与网格关联起来，即 $k=i\cdot S+j$。于是对于每个网格，我们就可以直接得到其最终的实例分割结果。然后我们将所有的网格结果汇总起来，就可以得到该图像的实例分割结果。最后，我们使用 NMS 来得到最终的实例分割结果，而不再需要其他的后处理操作。

3.2. 网络结构

SOLO 加在一个卷积主干网络后面。作者使用了 FPN，它输出一个特征金字塔，每个层级的特征图大小不同，通道数固定（通常 256 维）。这些特征图作为每个预测 head 的输入：语义类别和实例掩码。对于不同的层级上，head 的权重共享。在不同的金字塔中，网格数可能不同。

为了证明SOLO的通用性和有效性，作者构建了多个 SOLO 结构。不同之处在于：（a）特征提取的主干结构，（b）计算实例分割结果的网络 head，（c）训练的损失函数。大多数的实验都是基于下图中的 head 结构。为了进一步研究其泛化能力，作者也使用了其它变体。我们可以发现，实例分割 head 结构很直接、简单。除了输出卷积，其他卷积都是 $3\times 3$，分类分支的 Align 代表的是自适应性池化、插值或者 region-grid 插值，这三种的效果都差不多。

对于每一个 FPN，后面都会接一个上述的分支结构（SOLO head），从而实现多尺度预测。

3.3. SOLO 训练

3.3.1 Label Assignment

在类别预测分支中，网络需要给出 $S\times S$ 个网格的物体类别概率。具体点就是，只有当第 $(i,j)$ 个网格和某 ground truth 掩码的中心区域的重叠大于阈值时，它才被认为是正样本，否则就是负样本。 给定 ground truth 掩码的中心点 $(c_x, c_y)$，宽度 $w$，高度 $h$，通过缩放乘数 $\epsilon$：$(c_x,c_y,\epsilon w,\epsilon h)$ 来控制中心区域大小，中心点不是边框的中心，而是质心，其中 $\epsilon=0.2$，每个 ground truth 掩码上平均有3个正样本。

除了实例类别的标签，作者对每个正样本也进行了二值分割。因为每张图像有 $S^2$ 个网格，我们也有 $S^2$ 个输出掩码。对于每个正样本，我们标注对应的二值掩码。人们可能会担心，掩码的顺序会影响掩码预测分支，但是，作者证明最简单的行优先顺序就可以很好地解决。

3.4. Inference

SOLO 的推理步骤很直接。给定一张图片，我们将其传递入主干网络和 FPN，获得网格 $(i,j)$ 的类别得分 $p_{i,j}$，以及对应的掩码 $m_k$​，其中 $k = i \cdot S + j$。首先用置信度阈值 0.1 过滤掉分数比较低的预测，然后选取前 500 个排好序的 mask 进行 NMS 操作，然后用 0.5 的阈值进行 mask 二值化，保留前 100 个 instance mask 进行评估。

SOLO v2

SOLOv2 改进了 mask 分支，将其解耦为 mask kernel 分支和 mask feature 分支。另外，还提出了一种 Matrix NMS（非最大抑制）可以一次完成具有并行矩阵运算的 NMS。

Mask 分支的改进

上图中（a）和（b）为 v1 中提出的方法，（c）为 v2 提出的方法。

Vanilla head，如图中的（a）所示：以特征 F（维度为 $H\times W\times E$）做输入，输出 M（维度为$H\times W\times S^2$）。卷积核设为 G，G 的维度为 $1\times1\times E\times S^2$。F 和 G 进行卷积操作，作用是生成 $S\times S$ 个分类器，每个分类器都用于判断像素点的类别。

包含 $S\times S$ 输出通道的 M 是冗余的，因为在大多数情况下物体在图片中的分布都是稀疏的，起到作用的通道（分类器数目）远远少于 $S\times S$。Decoupled head，如图中的（b）所示，通过将这些分类器分解成 X 和 Y 两个方向，将输出通道从 $S\times S$ 降到了 $S+S$。

Dynamic head 更加直接，如图中的（c）所示。由于 F 是固定的，可以通过直接学习卷积核 G 来自动生成 M。SOLO v2中的 mask 分支被分解为 mask kernel 分支和 mask feature 分支，分别对应卷积核的学习和特征的学习。两个分支的输出最终组合成整个 mask 分支的输出。

Mask Kernel Branch

如图（c）所示，mask kernel 分支用来学习卷积核，输出为卷积核 $S\times S\times D$，其中 $S$ 是划分的网格数目，$D$ 是卷积核的通道数。Mask kernel 分支生成了 $S^2$ 个卷积核，每一个卷积核都包含 $D$ 个参数。对于 $1\times 1$ 的卷积，需要满足 $D=E$，即核大小为 $1\times 1\times E\times 1$。对于 $3\times 3$ 的卷积，需要满足 $D=9E$，即核大小为 $3\times 3\times E\times 1$。

Mask Feature Branch

由于 mask feature 和 mask kernel 被解耦分别进行预测，因此有两种构建 mask feature 的方式，一种是给 FPN 的每一个 level 都预测一个 mask feature，另一种是为 FPN 所有的 level 预测一个统一的 mask feature。这里采用的是第二种方式来提高效果和性能。

为了学习统一且高分辨率的 mask feature，通过不断地重复 $3\times 3$ 卷积、Group Norm、ReLU、$2\times 2$ 上采样的步骤，将 FPN 中 P2 到 P5 的不同特征合并为统一的输出。

Inference

在推理过程中，输入图像通过 backbone network 和 FPN 得到网格 $(i,j)$ 处的类别分数 $P_{i,j}$。首先使用置信度阈值 0.1 过滤低置信度的预测。然后使用相应位置的 mask kernel 对 mask feature 进行卷积。Sigmoid 操作后使用 0.5 为阈值将 soft mask 转换为二值 mask。最后一步是 Matrix NMS。
